# TCP 端口检测设计

## 目标

允许每台服务器选择检测方式，使关闭 `enable-status` 的 Minecraft 服务器仍可按端口连通性判定在线。

## 配置与兼容

- 为 `MonitorConfig` 增加 `checkMode`：`STATUS` 或 `TCP`。
- 既有已保存配置缺少该字段时，默认读取为 `STATUS`，保持原有行为。
- 新增服务器默认 `STATUS`。

## 检测行为

- `STATUS`：保留现有 Java 状态包请求，返回版本、玩家数、MOTD 和延迟。
- `TCP`：在相同 5 秒超时内建立 TCP 连接；连通即在线，详情为“端口可连接”，显示连接耗时；拒绝连接、超时或网络错误则离线。
- 手动检测与 WorkManager 自动检测共用同一按模式分发的检测函数。

## 设置界面

每张服务器设置卡增加“检测方式”下拉选择：

- 状态/MOTD 检测（默认）
- TCP 端口检测（适合关闭 enable-status）

## 验证

- 单元测试覆盖默认模式与旧配置反序列化兼容。
- 单元测试覆盖 TCP 模式可被分发。
- 运行全部单元测试和 Debug APK 构建。
